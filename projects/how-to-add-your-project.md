# How to Add Your Project

All Hack4Freedom Lagos 2026 projects are documented in this repository so they can live beyond the hackathon.

Each team is responsible for adding a short project summary to the `/projects` folder.

This helps:
- Showcase your work publicly
- Make your project easier to discover
- Support continued development, grants, and job opportunities

---

## What You Need to Do

Each team should add **one markdown file** describing their project.

You do not need permission — just follow the steps below.

---

## Step-by-Step Instructions

### 1. Fork the Repository

- Go to the Hack4Freedom Lagos 2026 repository
- Click **Fork** (top right)

---

### 2. Create Your Project File

Inside the `/projects` folder:

- Create a new file named after your project  
Example: 
lightningo.md
bitsave.md

---

### 3. Copy the Project Template

Copy the contents of `project-template.md` into your new file.

Fill in all relevant sections:
- Overview
- Problem
- Solution
- Technology stack
- Team members
- Links to your GitHub repo and demo
- Current status and next steps

You do not need to be perfect — clarity is more important than polish.

Example:
# LightningO

**Hack4Freedom Lagos 2026**

---

## Overview

LightningO is a platform that connects global sponsors with local builders through bounties, gigs, and grants using the Lightning Network and Nostr.

The goal is to create transparent, borderless opportunities for developers in Nigeria and the Global South.

---

---

### 4. Commit Your Changes

- Commit your new file with a clear message

---

### 5. Open a Pull Request

- Open a pull request back to the main repository
- Title it:  
Add <Project Name> project

That’s it 🎉

---

## Guidelines

- Projects must be **open-source**
- Link to the actual GitHub repository where your code lives
- Use clear, simple language
- This is documentation, not a pitch deck

---

## Need Help?

If you get stuck:
- Open an Issue in this repository
- Or ask for help in the Hack4Freedom community channel

Learning how to contribute documentation is part of the open-source experience.
